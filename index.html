Introdução

O Que É o Terminal

Um terminal é a interface mais fundamental entre você e seu computador. Enquanto a maioria das pessoas interage através de ícones e janelas, o terminal oferece uma forma diferente e muito mais poderosa: você se comunica através de texto, digitando comandos que o computador executa imediatamente.

Terminal vs. Interface Gráfica

Interface Gráfica (GUI)

Intuitiva, visual e amigável. Perfeita para tarefas do dia-a-dia. Mas esconde a maioria das opções avançadas — se queres fazer algo que a GUI não previu, ficas preso.

Terminal

Oferece CONTROLO TOTAL. Não estás limitado ao que alguém desenhou em uma interface. Podes fazer basicamente qualquer coisa que o sistema operativo permite, muito mais rápido.

Comparação Prática:
Na GUI: mover 1000 ficheiros demora 10 minutos de cliques. No Terminal: um único comando faz tudo em 3 segundos.

Por Que os Developers Usam Terminal

  • VELOCIDADE: Um dev profissional faz em 30 segundos no terminal o que levaria 5 minutos na GUI.
  • AUTOMATIZAÇÃO: Podes escrever scripts que executam tarefas automaticamente. A GUI não permite isto de forma prática.
  • TRABALHO REMOTO: Quando estás conectado num servidor via SSH, não tens interface gráfica. Só terminal.
  • PADRÃO DA INDÚSTRIA: Docker, Git, NPM, Kubernetes, AWS CLI — tudo isto é terminal. É OBRIGATÓRIO.

Linux, Terminal, Bash — Qual É a Diferença?

🏠 Linux é a CASA. Terminal é a PORTA. Bash é a LÍNGUA que falas quando entras.

  • Linux: Sistema Operativo, como Windows ou macOS. Usado em praticamente todos os servidores do mundo.
  • Terminal: A aplicação que abres para interagir com Linux. Como o Firefox para a internet.
  • Bash: A LINGUAGEM que falas dentro do terminal. "Bourne Again Shell" — o shell padrão em Linux.

Como Abrir e Sair do Terminal

  • Linux Desktop: Botão direito na área vazia do desktop → "Abrir Terminal Aqui"
  • Mac: Applications → Utilities → Terminal. Ou Command+Space e digita "terminal".
  • Windows 11: Windows Terminal ou PowerShell. Para Bash real, instala WSL.
  • Android (Termux): Instala o app Termux. Terminal completo no teu telefone.
usuario@computador:~$

Para sair: digita exit, logout, ou pressiona Ctrl+D.

Capítulo 1

Navegação pelo Sistema

Um desenvolvedor passa cerca de 40-50% do seu tempo no terminal, navegando pelo sistema de ficheiros. Dominar navegação é a base de tudo.

A Analogia da Casa: O teu computador é uma CASA GIGANTE. Cada CÔMODO é uma PASTA. Cada OBJETO é um FICHEIRO. O terminal é como te mover por esta casa sem corpo — só com palavras.

Bloco 1: Descobrir Onde Estás — 5 Comandos

pwd Print Working Directory

Mostra o caminho COMPLETO da pasta onde estás agora. É o teu «GPS» do terminal. Por trás, é uma variável de ambiente (PWD) que o próprio shell actualiza sempre que mudas de pasta.

$ pwd /home/maria/Documentos/Projetos/website
Pro tip: pwd > localizacao.txt salva o teu diretório actual num ficheiro para referência posterior.

pwd -P Caminho Físico Real

Mostra o caminho FÍSICO, ignorando links simbólicos. Útil quando trabalhas com atalhos ou estruturas complexas. Sem o -P, o pwd normal pode mostrar o caminho do link em vez do destino real — a diferença conta quando precisas da localização exacta no disco.

echo $PWD Variável de Ambiente

Alternativa a pwd usando uma variável de ambiente. Mais eficiente em scripts bash — lê uma variável em vez de executar um comando. Como é apenas uma variável, também podes usá-la dentro de outros comandos para construir caminhos dinâmicos.

$ echo "Estou em $PWD" Estou em /home/maria/Documentos

whoami Quem és tu no sistema

Mostra o utilizador actualmente logado. Essencial em servidores com múltiplos utilizadores ou quando trabalhas com permissões. É equivalente a executar id -un, mas mais rápido de escrever quando só precisas do nome do utilizador.

Atenção: Se whoami mostrar "root", tens privilégios de administrador. Usa com muito cuidado.

hostname Nome do Computador

Mostra o nome da máquina. Quando usas SSH para ligar a múltiplos servidores, hostname diz-te qual estás a usar AGORA. Em muitos sistemas, hostname -I também mostra o endereço IP da máquina na rede local.

SEGURANÇA CRÍTICA: Developers profissionais SEMPRE verificam hostname antes de executar comandos destrutivos. Um rm -rf no servidor de produção em vez de teste pode ser catastrófico.
Capítulo 1 — Bloco 2

Listar e Ver Conteúdo

12 variações do comando ls — o mais usado no terminal. Cada variação revela uma camada diferente de informação.

ls Listar ficheiros

Lista os ficheiros e pastas no diretório actual. Simples, directo. Não mostra ficheiros ocultos (os que começam com ponto). Por padrão, os resultados saem ordenados alfabeticamente; para outra ordem, precisas de combinar com flags como -t ou -S.

ls -l Formato Longo

Listagem detalhada com permissões, dono, tamanho e data de modificação. A primeira letra da linha (como o d em drwxr-xr-x) já diz se é um directório, um ficheiro normal ou um link.

drwxr-xr-x 2 maria grupo 4096 Jun 10 14:32 Documentos

ls -la Tudo + Detalhado

Combina -l (detalhado) com -a (mostrar tudo, incluindo ocultos). A maioria dos devs usa este como padrão. No Linux, ficheiros ocultos não têm nada de especial na estrutura — só começam por um ponto, como .bashrc ou .git.

ls -lh Tamanhos Legíveis

Igual a ls -l mas mostra tamanhos em K, M, G em vez de bytes. Muito mais fácil de ler. O h vem de \"human-readable\" — a mesma lógica aparece noutros comandos, como du -h e df -h.

$ ls -lh -rw-r--r-- 1 maria 150M Jun 10 arquivo.zip

ls -lS Ordenado por Tamanho

Lista ordenado por tamanho, maiores primeiro. Perfeito para encontrar o que está a ocupar espaço em disco. Combinado com | head -10, mostra rapidamente só os 10 maiores ficheiros de uma pasta cheia.

ls -lt Ordenado por Tempo

Ordenado por data de modificação, mais recente primeiro. Ideal para ver o que mudou recentemente. É normalmente o primeiro comando que muitos devs correm depois de puxar um repositório, só para ver o que mudou.

ls -ltr Ordem Cronológica Reversa

Igual a -lt mas invertido — mais antigos primeiro, mais recentes no final (onde podes ver). Muitos devs usam isto como padrão. É especialmente útil em pastas de logs, onde queres ver a entrada mais recente logo no fundo do ecrã.

ls -1 Um por Linha

Cada ficheiro numa linha separada. Ideal para processar a saída em scripts. Sem esta flag, o ls tenta organizar a saída em colunas quando corre num terminal interactivo, o que dificulta o processamento automático.

$ ls -1 | wc -l # conta o número de ficheiros

ls -d Apenas Directórios

Mostra apenas pastas, ignorando ficheiros. Útil quando tens centenas de ficheiros mas só queres ver a estrutura de pastas. Combina bem com wildcards — ls -d */ mostra só as pastas de topo, sem entrar em nenhuma delas.

ls -R Recursivo

Lista recursivamente toda a árvore de directórios. Cuidado com directórios com muitos ficheiros — pode ser muito lento. Em projectos grandes, é comum combinar com | less para poderes percorrer a saída com calma.

ls --group-directories-first

Mostra todas as pastas primeiro, depois os ficheiros. Torna muito mais fácil ver a estrutura de pastas. Não é um comando POSIX padrão — é uma extensão do GNU ls, por isso pode não existir em todos os sistemas Unix.

Alias útil: Muitos devs adicionam ao seu .bashrc: alias ll='ls -lah --group-directories-first' para ter sempre a melhor visão de um directório.
Capítulo 1 — Bloco 3

Navegar Estrategicamente

10 comandos para te moveres pelo sistema de ficheiros com velocidade e precisão.

cd Change Directory

O comando para entrar numa pasta. Podes usar caminhos relativos (a partir de onde estás) ou absolutos (a partir da raiz /). Sem qualquer argumento, cd sozinho leva-te sempre para a tua pasta home, tal como cd ~.

$ cd Documentos # caminho relativo $ cd /var/log # caminho absoluto $ cd # volta para home

cd .. Subir um nível

.. significa «pasta pai». Sobe um nível na hierarquia. Um dos gestos mais repetidos no terminal. Para saltos mais longos, podes encadear vários níveis numa linha só, como cd ../../../.

$ cd ../.. # sobe 2 níveis de uma vez

cd ~ Pasta Home

~ é um atalho especial para a tua pasta pessoal. Quando estás perdido em estruturas complexas, cd ~ traz-te de volta a casa. O til é expandido pelo próprio shell antes de chegar ao comando, por isso funciona da mesma forma com outros comandos, como cp ~/ficheiro.txt .

cd / Raiz do Sistema

/ é o topo absoluto da hierarquia. De aqui, vês bin, etc, home, var — todas as pastas principais do Linux. É também o ponto de partida de qualquer caminho absoluto — todo o sistema de ficheiros do Linux vive dentro desta única raiz.

cd - Directório Anterior

Volta para o directório onde estavas ANTES. Como um Ctrl+Z de navegação — vai e volta entre dois lugares rapidamente. Por trás, o shell guarda esse directório anterior na variável OLDPWD, que é o que o cd - consulta.

$ cd /var/log $ cd ~/Projetos $ cd - # volta para /var/log

cd ~utilizador Home de Outro Utilizador

Navega para a pasta home de outro utilizador (se tiveres permissão). Útil em servidores partilhados. Se não tiveres permissão de leitura nessa pasta, o comando ainda te deixa entrar, mas o ls seguinte não mostrará o conteúdo.

pushd Guardar e Mover

Move para um directório MAS guarda o anterior numa «pilha». Podes fazer pushd várias vezes e depois usar popd para voltar em ordem inversa. É literalmente uma estrutura de pilha — o último directório empilhado é sempre o primeiro a sair.

popd Voltar da Pilha

Remove o topo da pilha de directórios e volta para ele. Perfeito quando trabalhas com múltiplos directórios. Se a pilha estiver vazia e tentares popd outra vez, o bash mostra um erro a avisar que não há mais nada para tirar.

dirs Ver a Pilha

Mostra todos os directórios guardados com pushd. dirs -v mostra a pilha numerada — podes fazer cd +1 para saltar para o segundo. dirs -c limpa toda a pilha, útil quando queres recomeçar um workflow de navegação do zero.

Workflow Pro: Muitos devs criam o alias cd='cd && ls' para listar automaticamente após cada cd. Assim, cada navegação já mostra o conteúdo.
Capítulo 1 — Bloco 4

Encontrar e Buscar

12 comandos para localizar qualquer coisa em qualquer parte do sistema — por nome, tipo, tamanho ou data.

find Busca Recursiva

O comando mais poderoso para busca. Percorre TODO o directório e subpastas com critérios específicos. É mais lento que locate porque percorre o disco em tempo real, mas por isso mesmo encontra sempre resultados actualizados.

$ find ~ -name '*.txt' # todos os .txt na home $ find / -name 'config*' -type f # ficheiros começando com config

find -name Buscar por Nome

A flag mais comum. Usa wildcards: * (qualquer coisa), ? (um caractere). Use -iname para ignorar maiúsculas/minúsculas. As aspas à volta do padrão são importantes — sem elas, é o próprio shell que tenta expandir o wildcard antes do find o receber.

$ find ~/Projetos -name 'package.json' $ find . -iname '*.JS' # encontra .js e .JS

find -type Filtrar por Tipo

f = ficheiro, d = directório, l = link simbólico. Fundamental para não misturar ficheiros e pastas nos resultados. Também existem -type s para sockets e -type p para pipes nomeados, ainda que sejam usados com muito menos frequência.

$ find . -type d -name 'node_modules' # encontra todas as pastas node_modules

find -size Buscar por Tamanho

+100M (maior que), -10M (menor que). Quando o disco está cheio, este comando encontra os culpados. Sem o sinal + ou -, find -size 100M procura ficheiros com EXACTAMENTE esse tamanho, o que raramente é o que se quer.

$ find ~ -size +500M # ficheiros maiores que 500MB $ find . -type f -size +1G -name '*.mp4' # vídeos grandes

find -mtime Buscar por Data

-mtime -1 (modificado hoje), -mtime +30 (não modificado há mais de 30 dias). Essencial para limpeza de ficheiros antigos. Existem também -atime (último acesso) e -ctime (mudança de metadados), cada um a medir uma coisa ligeiramente diferente.

find -newer Mais Recente que Outro Ficheiro

Encontra tudo modificado depois de um ficheiro específico. Perfeito para ver o que mudou desde o último backup. O ficheiro de referência nem precisa de estar no mesmo directório — só a data de modificação dele é que importa.

locate Busca Rápida

MUITO mais rápido que find porque usa um banco de dados pré-indexado. Mas não encontra ficheiros criados recentemente. updatedb actualiza o índice. Essa base de dados costuma actualizar-se sozinha uma vez por dia via cron, por isso ficheiros muito recentes ainda podem não aparecer.

which Caminho do Executável

Mostra qual ficheiro executável é usado quando digitas um comando. Quando tens múltiplas versões (python2 vs python3), which diz qual está activo. Segue sempre a ordem do PATH — se houver duas versões instaladas, mostra a primeira que o shell encontrar.

$ which python /usr/local/bin/python3

whereis Todos os Locais

Como which, mas mostra o executável, o source e o manual. Mais completo para diagnosticar instalações. Ao contrário do which, não depende do PATH — procura em localizações padrão do sistema, mesmo que o comando não esteja acessível directamente.

type Tipo de Comando

Diz se um comando é built-in do shell, alias, função ou ficheiro externo. Mais informativo que which. É útil para perceber porque é que um comando se comporta de forma inesperada — muitas vezes é porque um alias está a sobrepor-se ao binário original.

whatis Descrição Rápida

Mostra uma descrição de uma linha de um comando. Mais rápido que man quando só queres saber o que faz. Usa a mesma base de dados das páginas man, por isso, se o manual não estiver indexado, whatis também não encontra nada.

$ whatis ls ls (1) - list directory contents

Estrutura, Metadados e Combinações

Bloco 5: Entender Estrutura — 8 Comandos

file Tipo de Ficheiro

Analisa o CONTEÚDO real do ficheiro e diz o que é — não depende da extensão. Um executável malicioso renomeado para .txt? file revela a verdade. Faz isto lendo os primeiros bytes do ficheiro (o chamado \"magic number\"), que identificam o formato real independentemente do nome.

stat Informações Completas

Mostra TUDO sobre um ficheiro: inode, permissões, links, tempo de acesso, modificação e criação. Muito mais que ls -l. Os diferentes carimbos de tempo respondem a perguntas distintas: quando foi lido, quando foi editado e quando o dono ou as permissões mudaram.

du Uso de Disco

Mostra quanto espaço um DIRECTÓRIO está a usar. du -sh * mostra o tamanho de cada item no directório actual. Por padrão, du soma também o espaço das subpastas inteiras, por isso os números podem parecer maiores do que esperarias à primeira vista.

$ du -sh . # tamanho total do projecto $ du -sh /* # tamanho de cada pasta na raiz

df Espaço Livre no Disco

Mostra quanto espaço LIVRE existe em cada partição. df -h mostra em formato legível. Quando atinge 90%, é hora de limpar. Ao contrário do du, que mede ficheiros, o df olha para a partição inteira — por isso os dois às vezes parecem não bater certo.

tree Visualizar Hierarquia

Mostra a estrutura de directórios visualmente em árvore. Muito mais legível que ls -R. tree -L 3 limita a 3 níveis de profundidade. Nem todas as distribuições Linux trazem o tree pré-instalado — muitas vezes é preciso instalá-lo com o gestor de pacotes (apt, dnf, etc.).

$ tree -L 2 -I 'node_modules|.git'

realpath Caminho Absoluto Real

Mostra o caminho absoluto e real, resolvendo links simbólicos. Em scripts, usa realpath para garantir a localização correcta. É particularmente útil dentro de scripts que precisam de saber a sua própria localização exacta, independentemente de onde foram chamados.

readlink Ler Link Simbólico

Mostra para onde um link simbólico aponta. readlink -f resolve links em cadeia (link a apontar para link). Se um link apontar para outro link, que por sua vez aponta para mais um, o -f resolve toda essa cadeia de uma só vez.

Bloco 6: Combinações Avançadas — 3 Padrões Poderosos

A Magia Real: Estes não são comandos únicos — são PADRÕES que transformam alguém que sabe comandos em alguém que RESOLVE PROBLEMAS.

find + grep Buscar Dentro de Ficheiros

find localiza ficheiros. grep filtra por conteúdo. Juntos, encontram conteúdo específico dentro de qualquer ficheiro. Este padrão é tão comum que muitas equipas o transformam num alias no .bashrc, poupando segundos preciosos todos os dias.

$ find . -type f -name '*.js' | xargs grep 'TODO' # Encontra todos os comentários TODO em todos os .js

ls + Wildcards Padrões de Correspondência

* (qualquer coisa), ? (um caractere), [] (conjunto). Wildcards funcionam com QUALQUER comando — ls, rm, cp, mv. É o próprio shell que expande os wildcards antes de o comando correr — por isso até comandos muito simples funcionam bem com eles.

$ ls *.txt # apenas .txt $ rm *.log # PERIGO: apaga todos os .log! # Sempre faz ls *.log primeiro para ver o que vai apagar

cd + ls Workflow de Exploração

O padrão fundamental de qualquer developer: entrar numa pasta e imediatamente listar o conteúdo. Repetido dezenas de vezes a dia. Com o tempo, este workflow torna-se automático — as mãos digitam cd && ls quase sem pensar, como um reflexo.

cd projecto && ls -la # entra e lista tudo de uma vez tree -L 2 # visão geral antes de navegar
